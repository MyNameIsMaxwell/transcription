# Транскрибация аудио

Веб-приложение для транскрибации аудиозаписей встреч с автоматической генерацией структурированного резюме через DeepSeek API.

## Возможности

- **Два движка транскрибации:**
  - Whisper (локально, CPU) — через faster-whisper
  - Yandex SpeechKit (облако)
- **Структурированное резюме** — тема, решения, задачи, ключевые тезисы
- **Экспорт в Word** (.docx) с читаемыми именами файлов
- **Отслеживание прогресса** в реальном времени
- **История обработок** для каждого пользователя
- **Drag-and-drop** загрузка файлов

## Системные требования

### Минимальные (для модели Whisper `tiny`)

| Параметр | Значение |
|----------|----------|
| CPU | 2 ядра |
| RAM | 2 GB |
| Диск | 5 GB (ОС + зависимости + модель) |
| ОС | Ubuntu 20.04+ / Debian 11+ |
| Python | 3.10+ |
| Сеть | Доступ к api.deepseek.com |

Подходит для обработки коротких записей (до 15 минут). Транскрибация будет медленной — примерно 2-4x от длительности аудио.

### Рекомендуемые (для модели Whisper `base` / `small`)

| Параметр | Значение |
|----------|----------|
| CPU | 4 ядра |
| RAM | 4 GB (8 GB для модели `small`) |
| Диск | 10 GB |
| ОС | Ubuntu 22.04 / Debian 12 |
| Python | 3.10+ |
| Сеть | Доступ к api.deepseek.com |

Комфортная обработка записей до 60 минут. Скорость транскрибации — примерно 0.5-1.5x от длительности аудио.

### Примечания по ресурсам

- **Модели Whisper по RAM**: `tiny` ~1 GB, `base` ~1.5 GB, `small` ~2.5 GB, `medium` ~5 GB, `large` ~10 GB
- **GPU не требуется** — приложение работает на CPU (compute_type=int8)
- **Параллельная обработка**: каждая одновременная задача потребляет дополнительно ~1 GB RAM (по умолчанию 2 воркера)
- При использовании **только SpeechKit** (без Whisper) достаточно 1 GB RAM и 1 ядра CPU

## Установка

1. Клонируйте репозиторий:
```bash
git clone <url> && cd transcribation
```

2. Создайте виртуальное окружение:
```bash
python -m venv .venv
source .venv/bin/activate
```

3. Установите зависимости:
```bash
pip install -r requirements.txt
```

4. Установите FFmpeg:
```bash
# Ubuntu/Debian
sudo apt-get install ffmpeg

# CentOS/RHEL
sudo yum install ffmpeg

# macOS
brew install ffmpeg
```

5. Создайте файл `.env` на основе `.env.example`:
```bash
cp .env.example .env
```

Заполните обязательные переменные (как минимум `DEEPSEEK_KEY`).

## Конфигурация (.env)

### Обязательные

- `DEEPSEEK_KEY` — API-ключ DeepSeek для генерации резюме

### Whisper (локальная транскрибация)

- `WHISPER_MODEL_SIZE` — размер модели: `tiny`, `base`, `small`, `medium`, `large` (по умолчанию: `base`)
- `WHISPER_COMPUTE_TYPE` — тип вычислений: `int8`, `float16`, `float32` (по умолчанию: `int8`)
- `WHISPER_NUM_THREADS` — количество CPU-потоков (по умолчанию: `4`)

### Yandex SpeechKit (облачная транскрибация)

- `YANDEX_API_KEY` — API-ключ Yandex Cloud
- `YANDEX_FOLDER_ID` — ID папки в Yandex Cloud
- `YANDEX_LANG` — язык (по умолчанию: `ru-RU`)

### Опциональные

- `FFMPEG_PATH` — путь к ffmpeg, если не в PATH
- `UVICORN_RELOAD` — `true` для dev-режима с автоперезагрузкой

## Запуск

### Локально (разработка)

```bash
python main.py
```

### Продакшн (сервер)

```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --workers 1
```

Не используйте несколько uvicorn-воркеров (`--workers > 1`), так как статусы задач хранятся в памяти процесса.

Для работы через reverse-proxy (nginx):

```nginx
server {
    listen 80;
    server_name your-domain.com;

    client_max_body_size 500M;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 600s;
    }
}
```

Приложение будет доступно по адресу: http://localhost:8000

## Использование

1. Откройте приложение в браузере
2. Перетащите аудиофайл в зону загрузки (или выберите через клик)
3. Выберите движок транскрибации
4. При необходимости задайте свой промпт для генерации резюме
5. Нажмите «Загрузить и обработать»
6. Отслеживайте прогресс в панели статуса
7. Скачайте готовое резюме в формате .docx

## Устранение неполадок

**FFmpeg не найден** — убедитесь, что `ffmpeg -version` работает в терминале, или укажите путь через `FFMPEG_PATH`.

**Ошибка загрузки модели Whisper** — при первом запуске модель скачивается из интернета. Нужен доступ к huggingface.co и ~500 MB-3 GB свободного места.

**Медленная обработка** — используйте меньшую модель (`tiny`) или увеличьте `WHISPER_NUM_THREADS`. Для быстрой обработки используйте SpeechKit.

**Ошибка SpeechKit** — проверьте `YANDEX_API_KEY` и `YANDEX_FOLDER_ID` в `.env`.
