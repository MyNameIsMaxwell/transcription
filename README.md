# Проект транскрибации аудио

Веб-приложение для транскрибации аудиофайлов с последующей генерацией резюме через DeepSeek API.

## Возможности

- **Выбор движка транскрибации:**
  - **Whisper (локально, CPU)** - использует faster-whisper для локальной обработки на процессоре
  - **Yandex SpeechKit (облако)** - использует облачный API Yandex для распознавания речи

- **Автоматическая генерация резюме** на основе транскрипции через DeepSeek API
- **Экспорт результатов** в формате Word (.docx)
- **Отслеживание прогресса** обработки в реальном времени
- **История обработки** файлов для каждого пользователя

## Требования

- Python 3.10+
- FFmpeg (должен быть установлен и доступен в PATH, или указан через переменную окружения `FFMPEG_PATH`)
- Для Whisper: ~500MB-3GB свободного места на диске (в зависимости от размера модели)
- Для SpeechKit: валидный API-ключ и Folder ID от Yandex Cloud

## Установка

1. Клонируйте репозиторий или скачайте проект

2. Создайте виртуальное окружение:
```bash
python -m venv .venv
```

3. Активируйте виртуальное окружение:
   - Windows: `.venv\Scripts\activate`
   - Linux/Mac: `source .venv/bin/activate`

4. Установите зависимости:
```bash
pip install -r requirements.txt
```

5. Установите FFmpeg:
   - **Windows**: Скачайте с https://www.gyan.dev/ffmpeg/builds/, распакуйте и добавьте `bin` в PATH
   - **Linux**: `sudo apt-get install ffmpeg` (Ubuntu/Debian) или `sudo yum install ffmpeg` (CentOS/RHEL)
   - **Mac**: `brew install ffmpeg`

6. Создайте файл `.env` на основе `.env.example` и заполните необходимые переменные

## Конфигурация

Создайте файл `.env` в корне проекта со следующими переменными:

### Обязательные

- `DEEPSEEK_KEY` - API-ключ для DeepSeek (для генерации резюме)

### Для Whisper (локальная транскрибация)

- `WHISPER_MODEL_SIZE` - размер модели Whisper (по умолчанию: `base`)
  - Доступные варианты: `tiny`, `base`, `small`, `medium`, `large`
  - Рекомендации для CPU: `tiny` (быстро, меньше точность) или `base` (баланс)
- `WHISPER_COMPUTE_TYPE` - тип вычислений (по умолчанию: `int8`)
  - Варианты: `int8`, `int8_float16`, `float16`, `float32`
- `WHISPER_NUM_THREADS` - количество потоков (по умолчанию: `4`)

### Для Yandex SpeechKit (облачная транскрибация)

- `YANDEX_API_KEY` - API-ключ Yandex Cloud
- `YANDEX_FOLDER_ID` - ID папки в Yandex Cloud
- `YANDEX_LANG` - язык распознавания (по умолчанию: `ru-RU`)

### Опциональные

- `FFMPEG_PATH` - путь к исполняемому файлу ffmpeg (если не в PATH)

## Запуск

```bash
python main.py
```

Или через uvicorn напрямую:

```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

Приложение будет доступно по адресу: http://localhost:8000

## Использование

1. Откройте браузер и перейдите на http://localhost:8000
2. Выберите движок транскрибации (Whisper или SpeechKit)
3. Загрузите аудиофайл (поддерживаются форматы: mp3, wav, ogg, m4a и др.)
4. При необходимости укажите промпт для генерации резюме
5. Нажмите "Загрузить и обработать"
6. Дождитесь завершения обработки (прогресс отображается в реальном времени)
7. Скачайте готовый файл с резюме в формате .docx

## Устранение неполадок

### Ошибка "FFmpeg не найден"

Убедитесь, что FFmpeg установлен и доступен в PATH:
```bash
ffmpeg -version
```

Если команда не работает, укажите путь явно через переменную окружения `FFMPEG_PATH` в `.env`.

### Ошибка при загрузке модели Whisper

При первом запуске модель Whisper загружается автоматически из интернета. Убедитесь, что есть доступ к интернету и достаточно места на диске.

### Ошибка "Yandex SpeechKit требует YANDEX_API_KEY"

Если вы используете SpeechKit, убедитесь, что в `.env` указаны `YANDEX_API_KEY` и `YANDEX_FOLDER_ID`.

### Медленная обработка

- Для Whisper: используйте меньшую модель (`tiny` или `base`) или увеличьте `WHISPER_NUM_THREADS`
- Для SpeechKit: обработка зависит от скорости интернета и загрузки API Yandex
